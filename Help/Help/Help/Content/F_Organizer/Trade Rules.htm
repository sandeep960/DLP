Trade Rules

This section lets the user provide a name and description for the trade rule.

 

Name

Mandatory

It is a text-field to provide a name for the Trade Rules.
 

Description

Optional

It is text-field to provide a description.

 

Opening

 

Strategy

Mandatory            The user needs to select the strategy from the list:

 

The symbol next to the strategy is a guide to choose the right strategy. When you select Price, Risk, Volatility and Profit Potential, strategies based on you selection are displayed.
 

Min Reward %

This filed appears when strategies like Bull Call or Iron Condors are selected. The user specifies the reward % over risk. If you input 30%, position potential profit should be at least 30% of risk in the position.

 
 

Layers

This field appears when strategies like Layered Credit Spreads are selected. The user can select from 5 layers and also needs to input an Entry Stop which is the number of days before DTE.

Input number of layers that risk amount can be distributed. If you input 3 for $30,000 risk amount, system allocated risk of $10,000 for each layer.

 

 

 

Entry stop _____ Day(s) before DTE

 

Input number of days to stop opening new positions.

 

 

 

Report Context

Mandatory     This option lets the user select the style of the output of BackTesting.

 When analyzing results from backtest, there is a need to see different key performance indicators.

Example: For Layered Credit Spreads strategy, we see additional field with layers and corresponding risk-reward. For this case, Trend will be used the majority of the time.
 

Spread

This option has 3 choices – Market, Mid-Price and Limit.

Fills are assumed in back test mode. In live trading, order fulfillment is dependent on the market.
   

Market

Selecting market orders will assume selling at ask price and buying at bid price.
   

Mid-Price

This will buy and sell in between Bid and Ask prices.
   

Limit

To mimic close to live trading, you can input 60 in Buy and 40 in Sell.  In this example, trader is willing to pay 60% of the spread while buying and take 40% of the spread when selling.

 

Stops

This section lets the user provide an exit criteria.

 

 

Ignore Exit Dates

Selecting Yes will ignore the exit date created by exit criteria and only will exit based on expiration date of the options in the position.

Example: Trader creates entry when MACD turns bullish and exit dates when MACD turns bearish for the last 10 years.  For these entry position, if we enter into Long Call in trade rules and want to bypass the MACD turning bearish and continue in the trade until the trading day before option expires.

 
 

Stop after _____  Trading Days

Trade will exit after entered number of trading days from day of entry.

Example: If trader enters 10, trade will exit after 10 trading days from the day position is opened. 

 
 

Exit ____ Day(s) before Days to Expiration

Entering 10 in the day column will make the position close before 10 calendar days to expiration. 

 

Secondary Confirmation

This section lets the user select a secondary confirmation filter.

 

 

State

Select any of the states from the State Modeling.
 

Signal

Select a signal from the list of user defined signals.

 
 

Filter Action

If Delay Entry is selected, it will check every day and trade will be accepted only if we get confirmation before exit date from Entry-Exit dates.

If Reject is selected, trade will is accepted when this secondary filter is satisfied. If secondary filter is not satisfied, trade will be rejected.

 

Option Expiration

This section appears only after selecting a strategy. It provides options to choose your options expiration.

 

 

Options

Select weekly or monthly
 

State Modeling

You can manage this position using Conservative (T1), Moderate (T2) or Aggressive T3) on expected and unexpected directions. Selecting Yes will let you add these targets and stop loss to manage the position.

 
 

Days to Expire

If Manual is selected, you need to input min and max days.

E.g.: Entering 20 and 65 days will let the system select an expiration between 20 and 65 days and as close as 20 days. If an option expiration between these days does not exist for that specific entry point, trade will be rejected.

 

If State is selected, State Modeling algorithm will identify the average days that ticker may exist in its current state. When State is selected, it will use average days identified as min days and it will automatically add 30 days to max value for each entry point in Entry-Exit dates. You can also add additional days to both min and max values by entering number of days.

 

Strike Selection

This section along with Option Expiration appears only after selecting the strategy. You can input your strike selection.

There are 4 ways to select Strike Selection:

  • Delta
  • Minimum points from leg
  • Exact points from leg
  • Strikes from leg
Delta

Select Strike Selection Model as Strike.

User can select a range of delta between which the strike will be selected. Also, direction whether to find the strike on lower side or higher side of the range can be selected in the direction field.

Example: If 20 and 30 are selected in Min and Max with Direction as Min to Max is selected, strike between 20 and 30 with delta close to 20 will be selected. If 20 and 30 are selected in Min and Max with Direction as Max to Min is selected, strike between 20 and 30 with delta close to 30 will be selected.

 

Maximum points from leg

Select Strike Selection Model as Maximum points from leg.

Trader can select 10 points above or below current price. Points are different from percentage.

Example:

On 2/1/2016, if naked put has to be sold at 1 standard deviation below the current price for SPX, based on the above settings, strike will be selected when SPX closed at $1939.38.

Stage 1: Management Adjustments

This section appears only after selecting the strategy. It provides options to manage the positions based on reward or risk tolerance parameters.

As underlying price keeps moving which impacts options pricing in a positive or negative way to existing position, it is important to manage the position based on reward or risk tolerance parameters.

Winning portion of the management is when you have a trade which is in profits and losing portion is applicable for a trade which is in loss.

Click to expand the section.

 

 

Profit % >=

Input in this field indicates to manage the position when position makes more or loses more money than Profit% multiplied by second field Profit On. If you input 100% on Initial cost $1000, trade will be managed when it makes more profit than $1000.

 
 

Profit On

There are four possibilities:

  • Initial Cost - the cost to open a trade
  • Current Risk - is risk in the trade at any given time
  • MaxProfit - maximum potential profit in the trade
  • MaxRisk - maximum potential risk in the trade

 
 

Trailing Profit Trigger >=

With 20% in the trigger field and keeping at least 80% of profit, system will manage the trade when 80% of the maximum profit so far made after 20% profit.

 

Example: If you input 100% on Initial cost $1000, trade will be managed when it makes more profit than $1000, 20% trigger and keeping 80%, trade will be managed after it makes $200.

 

Underlying Price (%) >=

This field lets you manage the position when underlying moves greater or less than stated % from underlying close price from point of entry.

 
 

Position Delta >=

This field lets you manage a position when position delta reaches above or below this input value.

 
 

Leg / BrkEvn (Break-Even)

This field lets you can manage the position by Greeks for each leg or underlying meets or stays within or moves of break-even ranges depending on each strategy.

 

 

Adaptive Trade Management

If the winning and losing criteria is not met, then the symbol moves from each state to the selected stage. The default winning and losing criteria is close.

 

Example 1: Long Call strategy

In case of a long call strategy, the symbol moves from State 1 to Stage 4 if the winning and closing criteria is not met.

 

Example 2: Layered Credit Spread strategy

This trades the selected symbols through a 2 layered approach. When the inputted symbol moves from State 1 to State 4 then execute the selected strategy on the symbol in the layered credit spread.

The layered strategies are Bull Put, Bear Call and Iron Condor.

 

Example 3: Married Put

This trades the selected stock through a layered hybrid approach. The symbol will move from State 1 to stage 4 if it meets the provided criteria.
